## Session 2: NER Pipeline (evening)

### Objective
Build the NER pipeline to extract structured data from OCR-transcribed civil registry acts (abymes/1842).

### Steps

1. **Schemas** (`trocr_handwritten/ner/schemas.py`) — Pydantic models: ActRecord, PersonInfo, DeathActEntity, BirthActEntity, NERResult, flatten_ner_result()
2. **Dataset builder** (`trocr_handwritten/ner/dataset.py`) — Reconstructs acts from YOLO crop .md files using metadata.json reading_order. Continuation-merging logic (text without preamble appended to previous act). 228 acts from abymes/1842 (147 deaths, 80 births, 1 header).
3. **Regex extractor** (`trocr_handwritten/ner/regex_extractor.py`) — French civil registry patterns. Handles OCR variants (negresse/negritte/nigresse), French number conversion, habitation/owner/declarant/registration/occupation/officer extraction.
4. **LLM infrastructure** — Added text-only async methods to `llm/base.py` and `llm/providers/gemini.py` for function calling (additive only).
5. **LLM extractor** (`trocr_handwritten/ner/llm_extractor.py`) — Async Gemini extraction with function calling. Model: gemini-3-flash-preview (~$0.001/act).
6. **Comparison** (`trocr_handwritten/ner/compare.py`) — Field-by-field regex vs LLM comparison.
7. **Pipeline** (`trocr_handwritten/ner/pipeline.py`) — CLI: dataset → regex → LLM → compare.
8. **Merge** (`trocr_handwritten/ner/merge.py`) — Consolidates regex + LLM into single CSV ({field}_regex, {field}_llm, {field}_final). UTF-8 BOM + semicolon delimiter for Excel.
9. **Prompts** — `config/ner_death.prompt`, `config/ner_birth.prompt` with slave registry conventions.

### Key findings
- LLM extracts ~25-30% more owners than regex (uses Marge "au Sr X", understands "son habitation")
- LLM infers child sex from grammatical gender ("immatriculé" = homme)
- LLM resolves relative dates ("ce matin" → "01/01/1842")
- Updated prompts prevent hallucinated ownership from "habitant propriétaire"
- Rate limiting on Gemini at 10 concurrent — use 5 for full runs
