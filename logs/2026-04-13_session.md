### 2. Schema additions

**`family` field** added to owner clusters — groups owners sharing a surname even when treated
as distinct people (e.g. Caillou Junior + Joseph Caillou both family "Caillou";
Veuve Le Bourg Lacoudrai + Bramfeld Le Bourg Lacoudrai both family "Le Bourg Lacoudrai").

### 3. Plantation self-reference rule

When `habitation_name_raw` is "habitation de [owner X]" / "propriété de [owner X]" / etc.,
plantation canonical = owner's name (since the wording proves a habitation exists).
Vague references like "habitation de sa maîtresse" → resolved by looking up the owner field
in the same act and using that owner as the plantation name.

### 4. "Hon X" / "Bon X" / "Habon X" / "habitation X" patterns (anse, gosier)

These are LLM extraction artifacts where `habitation X` was misparsed as the owner. Treated
as anonymous-owner entries with `canonical=null` and `family=X` (the plantation name from the
suffix). When the suffix doesn't match the actual habitation field (e.g. "Habon La Joyeuse"
on hab "Beaufond"), kept in null+null cluster.
